# 창의적 문제해결 가이드

"창의/혁신/개선" 문항에 대응할 에피소드를 발굴·강화할 때 참조한다.

## 목차

1. 창의 문항이 실제로 묻는 3가지
2. "창의"에 대한 나만의 정의 만들기
3. TRIZ 기반 17가지 창의적 문제해결 원리
4. 에피소드에서 창의성 강화하는 체크리스트
5. 참신한 형식 활용 팁

---

## 창의 문항이 실제로 묻는 3가지

| # | 평가 요소 | 핵심 질문 |
|---|---|---|
| 1 | **주도성** | 문제를 남이 알려줘서가 아니라, 본인이 자발적으로 인식했는가? |
| 2 | **분석력** | 문제에 논리적/분석적으로 접근했는가? 감이 아니라 근거가 있는가? |
| 3 | **참신성** | 해결 방법이 기존과 다른가? 발상의 전환이 있었는가? |

> 기업이 원하는 "창의"는 천재적 발명이 아니다.
> **기존과 다른 방법론을 근거를 갖고 제시**한 것이면 충분하다.

---

## "창의"에 대한 나만의 정의 만들기

에피소드만 쓰면 사건에 매몰된다. **창의에 대한 나만의 한 문장**을 에피소드 앞뒤에 배치하면 인상이 깊어진다:

- "'참신함'이란 고객의 모든 오감을 만족시키고자 고민할 때 나오는 것입니다."
- "'상황'에 매몰되는 것이 아니라 '어떻게'할 것인지 고민했을 때 기존과는 다른 방안을 도출할 수 있었습니다."
- "'창의적 사고'는 상상력과 실험 의지, 기존의 것을 전복하는 태도에서 시작됩니다."

## TRIZ 기반 17가지 창의적 문제해결 원리

러시아 과학자 알트슐러가 특허 수백만 건을 분석해 정리한 40가지 원리 중, 자소서에 활용 가능한 17가지를 발췌:

| # | 원리 | 자소서 적용 예시 |
|---|---|---|
| 1 | **분할** | 120분 경기를 4쿼터로 나눠 전원 출전 기회 보장 |
| 2 | **분리** | 핵심 업무와 반복 업무를 분리하여 효율화 |
| 3 | **부분만 다르게** | 매대 일부만 타겟 고객 맞춤으로 변경 |
| 4 | **비대칭** | 시간대별로 다른 전략 적용 (아침/점심/저녁) |
| 5 | **통합** | 분산된 정보를 하나의 시트로 통합 관리 |
| 6 | **다용도** | 하나의 시스템이 여러 기능을 수행하도록 설계 |
| 7 | **포개기** | 단계별 목표를 포개어 점진적으로 달성 |
| 8 | **예방조치** | 문제 발생 전에 사전 점검 체계 구축 |
| 9 | **선행조치** | 발주량을 미리 산정하여 결품 방지 |
| 10 | **반대로** | "사지 마세요" 역발상 문구로 구매 유도 |
| 11 | **차원변경** | 2D 데이터를 시각화하여 인사이트 도출 |
| 12 | **해로움을 이롭게** | 실패 경험을 학습 자료로 전환 |
| 13 | **셀프서비스** | 수작업을 자동화 스크립트로 대체 |
| 14 | **대신하기** | 직접 방문 대신 화상회의/원격 시스템 도입 |
| 15 | **유연한 막** | 기존 시스템에 유연한 설정 레이어 추가 |
| 16 | **색상변경** | 시각적 구분(컬러코드)으로 정보 전달력 향상 |
| 17 | **매개변수 변화** | 제품 형태 변경 (삼계탕→닭죽으로 양 증가) |

## 에피소드에서 창의성 강화하는 체크리스트

에피소드를 인수분해한 후, 아래를 점검:

- [ ] **문제 인식이 주도적인가?** — 누가 시켜서가 아니라 본인이 발견한 문제인가?
- [ ] **분석 과정이 있는가?** — 감이 아니라 데이터/관찰/논리에 기반했는가?
- [ ] **기존과 다른 점이 있는가?** — 기존 방식 대비 무엇이 달라졌는가?
- [ ] **결과가 구체적인가?** — 정량적 또는 정성적 변화가 명확한가?
- [ ] **나만의 창의 정의가 있는가?** — 에피소드를 관통하는 한 문장이 있는가?

---

## 참신한 형식 활용 팁

창의 문항이라면 **내용뿐 아니라 형식도 차별화** 가능:
- 신문기사 형식
- 3인칭 시점 (인간극장 버전)
- 사내 인터뷰 형식
- Q&A 형식

단, 형식 차별화는 내용이 충분히 탄탄할 때만 시도할 것.
