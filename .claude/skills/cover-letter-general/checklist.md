# 사실관계 검증 체크리스트 (범용)

자소서 초안 완성 후, 모든 문항에 대해 아래 항목을 점검한다.

## 1. 경험 진위 및 톤 다운
- [ ] 이 경험은 지원자 본인이 직접 수행한 것인가?
- [ ] 팀 프로젝트에서 다른 팀원의 역할을 자신의 것처럼 서술하지 않았는가?
- [ ] "혼자 다 했다" 혹은 "완벽하게 구현했다"는 방어 불가능한 과장된 인상을 주지 않는가? (특히 복잡한 도메인/알고리즘 구현 시 유의)
- [ ] AI나 튜토리얼의 도움을 받은 코드를 순수 본인의 창작인 것처럼 속이지 않았는가? (면접에서 압박 질문의 대상이 됨)
- [ ] 프로젝트 인원 수, 기간, 팀 구성이 사실과 일치하는가?

## 2. 정량 수치
- [ ] 제시된 수치(매출, 개선율, 기간 등)는 측정 근거가 있는가?
- [ ] 수치의 전제 조건이 누락되지 않았는가?
- [ ] 면접에서 "이 수치의 근거가 무엇인가요?"라는 질문에 답변할 수 있는가?
- [ ] 과장이나 표현상의 왜곡은 없는가?

## 3. 직무 역량 및 기술 팩트 체크
- [ ] 언급한 기술/역량을 실제로 보유하고 있는가?
- [ ] 면접에서 해당 기술 스택에 관한 원리(Under the hood) 심화 질문에 답변할 수 있는가?
- [ ] (매우 중요) **실제 수강하지 않은 과목/교육**을 허위로 기재하지 않았는가? (AI 독학 경험은 실무 서적 독학 등으로 대체)
- [ ] **초기 설계(Architecture Design)** 경험과 사후 **리팩토링(Refactoring)** 경험을 혼용/혼동하여 기재하지 않았는가?
- [ ] 자격증/수료증 등 객관적 증빙이 필요한 내용은 사실과 부합하는가?

## 4. 기업 정보
- [ ] 기업명, 직급/호칭이 정확한가? (입사 vs 입행 vs 입회 등)
- [ ] 기업의 인재상/핵심 가치 인용이 최신 정보인가?
- [ ] 기업의 사업 영역 서술이 사실과 부합하는가?
- [ ] 경쟁사와 혼동할 수 있는 표현은 없는가?

## 5. 문항 의도 부합
- [ ] 문항이 묻는 역량과 실제 서술하는 역량이 일치하는가?
- [ ] "노력 경험"에 기술 자랑만 나열하거나, "팀워크"에 개인 성과만 쓰지 않았는가?
- [ ] 질문의 핵심 키워드(예: "가장 어려웠던", "창의적으로 해결한")에 정확히 응답하고 있는가?

## 6. 표현 품질
- [ ] 동일 키워드가 한 문항 내에서 3회 이상 반복되지 않는가?
- [ ] "~합니다/~했습니다" 등 종결어미가 3문장 이상 연속 반복되지 않는가?
- [ ] 부정적 단어(절망, 배신감, 경멸 등)나 비속어/은어(현타, 대박 등)를 사용하지 않았는가?
- [ ] 수동태("~되었습니다", "~하게 되었습니다") 대신 능동태로 작성했는가?
- [ ] 상황 설명이 2~3문장(3줄 이내)으로 요약되었는가?
- [ ] 행동의 구체성이 Level 3(HOW가 보이는 수준)에 도달했는가?
